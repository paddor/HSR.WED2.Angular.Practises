Installation & Expressions

Lesen Sie die folgenden Abschnitte von angularjsbook.com:

  • Kapitel: Basics/Setup
  • Kapitel: Basics/Expressions

  1. Laden Sie die neuste Version von AngularJS 1.4.x von der AngularJS Seite herunter (z.B. https://code.angularjs.org/1.4.3/angular.min.js) und erstellen Sie ein gültiges HTML-Dokument, in das Sie das Framework einbinden.
  2. Definieren Sie einen Container (div), den Sie als AngularJS App deklarieren. Benutzen Sie die vom HTML-Standard vorgeschlagene Variante dazu.
  3. Erstellen Sie eine Expression, die den String
    AngularJS ist ein Open-Source-Framework des US-amerikanischen Unternehmens Google Inc.
    als Titel ausgibt und dabei auch noch seine Länge angibt, z.B.:
    AngularJS ist ein Open-Source-Framework des US-amerikanischen Unternehmens Google Inc. (86 Zeichen)
    Deklarieren Sie dazu den String als Variable in einer Expression und verwenden Sie diese für beide Ausgaben.

    Siehe solution/installation.html.
    Beachten Sie die Verwendung von "data-ng-app" anstelle von "ng-app". Nach dem HTML5-Standard müssen nicht-standardisierte Attribute mit "data-" beginnen. Zudem wird mögliche Kollisionen mit zukünftigen HTML-Attributen vorgebeugt.

  4. Beantworten Sie dazu folgende Fragen:
    1. Warum empfiehlt es sich, AngularJS lokal einzubinden statt über einen Content Delivery Service wie z.B. Google Code?

      • Sie können auch ohne Netzwerkverbindung arbeiten.
      • Sie sind nicht auf die Zuverlässigkeit des Google Servers angewiesen.
      • Wenn Sie keine spezifische Version eingebunden haben sondern die jeweils Aktuelle, riskieren Sie, das Google diese aktualisiert und ihre Applikation plötzlich nicht mehr funktioniert.
      • Sie liefern keine Nutzungsdaten an Google.

    2. Eigentlich sollten JavaScript Dateien ganz unten im Dokument-Body eingebunden werden. Können Sie sich einen Grund vorstellen, weshalb wir AngularJS im Header einbinden?

      Wird AngularJS als letztes eingebunden, so passiert folgendes: Der Browser parst bereits während dem Laden des HTML Dokuments dessen Inhalt. Als erstes werden die Angaben im Head eingelesen und referenzierte Scripts und Styles nachgeladen. Dann wird der Body geparst. Der Browser parst und rendert den Inhalt der HTML Seite, bevor er durch AngularJS verarbeitet werden kann. Sobald AngularJS geladen wurde, rendert dieses die Expressions und Directives und ersetzt den Inhalt. Dadurch wird ein unschöner "Flash" erzeugt, bei dem ganz kurz das der Source Code des Templates angezeigt wird.

      Siehe auch http://stackoverflow.com/questions/15538125/angularjs-in-head-vs-body

Scopes

Lesen Sie die folgenden Abschnitte von angularjsbook.com:

  • Kapitel: Basics/Scopes

  • Sie haben bereits bei den letzten Aufgaben Scope Variablen verwendet: nämlich die Variable "events". Alle im Scope definierten Variablen stehen im Template zur Verfügung und können auch modifiziert werden.
  • Erweitern Sie das Beispiel mit den Anlässen um die Möglichkeit, neue Anlässe hinzuzufügen. Benutzen Sie die Vorlage template/scopes1.html. Gehen Sie dabei wie folgt vor:
    1. Entfernen Sie den Filter für ng-repeat, der alle Orte herausfiltert, die nicht Rapperswil heissen, um Verwirrung beim Ausprobieren zu verhindern.
    2. Ergänzen Sie den Scope um ein "leeres"-Event-Objekt.
    3. Binden Sie die Eigenschaften dieses Objektes an die Eingabefelder.
    4. Definieren Sie für den Reset-Button eine Aktion, sodass beim Klicken das Event-Objekt zurückgesetzt wird.
    5. Definieren Sie für den Submit-Button eine Aktion, sodass beim Klicken der neue Event der Liste der Events hinzugefügt wird und das Event-Objekt zurückgesetzt wird.
    6. Passen Sie den Preview-Container so an, das er nur angezeigt wird, wenn eines der Felder einen Wert enthält und geben Sie im enthaltenen Textblock (span) eine Vorschau des Eventes aus.
    Tipp (Versuchen Sie es zuerst ohne)

    Siehe solution/scopes1.html.
